## Overview

The Wrkstrm brand represents a transition from a decade of legacy CLI tools into a sophisticated, unified ecosystem of agentic assistants and type-safe software artifacts. This document codifies the visual, documentation, and technical identities of the project.

## 1. Visual Identity (2026+ Evolution)

The visual brand has moved from the legacy "Original Identity" (2015-2025) to a new, tool-centric direction.

- **Selected Concept:** "Terminal Window" (`concept-terminal-block-v2.png`)
    - *Description:* A sophisticated mark featuring the "inverted W" signature.
    - *Role:* Represents the "Future (2026+)" of the Wrkstrm project.
- **Color Palette:**
    - Primary Accent: Blue (`#0071e3`)
    - Backgrounds: Light gray (`#f5f5f7`) & White (`#ffffff`)
    - Typography: System fonts (San Francisco/Segoe UI).

## 3. Agent Personas (The "Team")

The "brand" of the CLI assistant is partitioned into specialized personas, each with its own operational mandate:

- **Codex:** The default "Swiss-army-knife" builder.
- **Clia:** The orchestrator of rituals and safe workflows.
- **Cadence:** The Project Manager (governance & rhythm).
- **Carrie:** The Documentation Steward.
- **Patch:** The Incident Responder.

## 4. Technical Identity (Identifiers)

We enforce brand consistency at the code level by standardizing software identifiers using reverse-DNS mapping.

- **Entity Presets:**
    - `personal` -> `me.rismay.<product>.<platform>.<surface>.<lane>`
    - `llc` -> `studio.laussat.<product>.<platform>.<surface>.<lane>`
    - `inc` -> `app.wrkstrm.<product>.<platform>.<surface>.<lane>`
- **Infrastructure Surfaces:**
    - `menubar` (Status Menu)
    - `daemon` (Background Services)
