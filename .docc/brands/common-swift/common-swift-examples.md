## Purpose

Concrete do/don't examples for applying the CommonSwift brand in docs and UI.

## Do

- Use the primary color for a single key CTA per view.
- Keep long-form docs on neutral surfaces with high-contrast text.
- Use consistent spacing and predictable grid rhythm.

## Avoid

- Multiple competing accents on the same page.
- Dense layouts that reduce readability.
- Low-contrast text on colored surfaces.

## Sample layouts

- Documentation page: centered content column, left navigation, minimal UI chrome.
- Tooling dashboard: left navigation, primary content, right inspector only when needed.

## Code samples

- Use monospaced blocks with clear boundaries.
- Prefer short, focused snippets with explicit titles.
