## Core components

- LibrarySidebar
- DocCReaderView
- ProgressInspector
- NotesPanel
- ExercisePanel
- TestResultsList
- BookOutlineView
- PracticeSetView

## Component rules

- Buttons: primary uses baby blue; secondary uses outline.
- Panels: neutral surfaces with subtle borders.
- Code blocks: monospaced with clear background contrast.

## States

- Focus: accent blue ring.
- Error: red with clear icon + message.
- Success: green with brief confirmation.

## Do

- Keep component variants minimal.
- Use consistent spacing between components.

## Avoid

- Overloading components with multiple accent colors.
- Hidden state changes without visible cues.
